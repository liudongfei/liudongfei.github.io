Linux运维—启动流程

Posted on 2023-04-13 | Post modified: 2023-04-13 | In linux
  1. 加载BIOS的硬件信息与进行自我测试，并依据设置取得第一个可启动的设备；
  2. 读取并执行第一个可启动设备内MBR的bootloader（即是grub.spfdisk等程序）;
  3. 依据bootloader的设计加载Kernel，Kernel会开始检测硬件与加载驱动程序；
  4. 在硬件驱动成功后，Kernel会主动调用init进程，而init会取得run-level信息；
  5. init执行/etc/rc.d/rc.sysinit 文件来准备软件执行的操作环境（如网络、时区等）；
  6. init执行run-level的各个服务的启动（script方式）；
  7. init执行/etc/rc.d/rc.local文件；
  8. init执行终端机模拟程序mingetty来启动login进程，等待用户登陆；

Linux运维—软件管理

Posted on 2023-04-13 | Post modified: 2023-04-13 | In linux

两大主流软件管理工具：RPM（Redhat/Fedora）和DRPM（Debian/Ubuntu）
RPM默认安装的路径：

  • /usr/bin：一些可执行文件；
  • /usr/lib：一些程序使用的动态函数库；
  • /usr/share/doc：一些基本的软件使用手册和帮助文档；
  • /usr/share/man：一些man page文件；

RPM常用的操作命令：

  • rpm -ivh pk_name：安装软件；
  • rpm -Uvh pk_name：安装或升级软件；
  • rpm -Fuh pk_name：更新软件；
  • rpm -q pk_name：查询软件是否安装；
  • rpm -qa：列出所有的已经安装在本机系统上的软件；
  • rpm -qi pk_name：列出该软件的所有信息；
  • rpm -ql pk_name：列出该软件的所有文件和目录所在的完成文件名；
  • rpm -qc pk_name：列出该软件的所有配置文件；
  • rpm -qd pk_name：列出该软件的所有帮助文件；
  • rpm -qR pk_name：列出该软件有关的所有依赖软件所含的文件；
  • rpm -e pk_name：卸载该软件；

YUM是通过解决RPM的安装/删除/更新的依赖工具:

  • yum search pk_name：搜索该软件；
  • yum list：列出yum server上提供的所有软件名称；
  • yum info pk_name：查看该软件的功能信息介绍；
  • yum list updates：列出目前yum server上可供本机进行更新的软件；
  • yum provides command_name：列出提供该命令的软件有哪些；
  • yum install pk_name：安装该软件；
  • yum update pk_name：更新该软件；
  • yum remove pk_name：卸载该软件；
  • yum repolist all：列出所有的yum server

YUM的配置文件：/etc/yum.repos.d/*.repo

Linux运维—系统日志

Posted on 2023-04-13 | Post modified: 2023-04-13 | In linux

系统常见的日志文件名

  • /var/log/dmesg：系统启动时内核检测日志，dmesg命令利用的就是这个文件；
  • /var/log/lastlog：记录系统上所有的帐号最近一次登陆的相关信息，lastlog命令利用的是这个文件；
  • /var/log/messages：记录系统发生的所有错误信息，这个文件非常重要，发生错误一定要查这个文件；
  • /var/log/secure：记录涉及到帐号密码登陆的信息都会记录在这个文件中；

系统的日志记录相关的进程

  • syslogd：主要记录系统与网络等服务的信息，配置文件/etc/syslog.conf；
  • klogd：主要记录内核产生的各项信息,；
  • logrotate：主要进行日志文件的轮替功能，配置文件/etc/logrotate.conf。

Linux运维—用户管理

Posted on 2023-04-13 | Post modified: 2023-04-13 | In linux

用户ID的范围

  • 0：为系统管理员。
  • 1～499：为保留给系统使用的ID。这些系统帐号通常是不可登陆的。
  • 500～65535：为普通用户使用的ID。

用户相关的配置文件

/etc/passwd：用户帐号文件
/etc/shadow：用户密码文件
/etc/group：用户组文件

用户管理

添加用户：
useradd username：添加用户
passwd username：用户密码
userdel -r username：删除用户

用户身份切换

  • su - username：切换身份，使用login-shell的变量文件读取方式登陆，如果不加“-”则使用nologin-shell的方式登陆，此时bash的环境变量不会改变，如果username省略，则代表切换为root命令，切换命令需要密码；-c代表用切换的身份执行的命令；
  • sudo -u username：欲切换的用户，若没有这个参数u代表切换的身份为root用户；-c代表用切换的身份执行的命令；

通过编辑/etc/sudoers文件，添加可以执行sudo权限的用户，或者用户组（最左边加%）。
文件格式：用户帐号 来源主机名=（可切换的身份） 可执行的命令

Linux运维—Bash Shell的常见功能

Posted on 2023-04-13 | Post modified: 2023-04-13 | In linux

bash shell的变量功能

  • 环境变量：用env查看环境变量
  • 自定义变量：用set查看所有变量

环境变量和自定义变量的区别在于，环境变量可以被子进程继承，自定义变量则不可以。可以使用export将自定义变量转为环境变量。ulimit限制用户所使用的资源限制，ulimit -a列出目前用户的资源限制。

bash shell的环境配置

login-shell的环境配置文件加载流程依次如下：

  • /etc/profile：系统整体的设置，不建议修改这个文件
  • ～/.bash_profile：用户个人配置
  • ～/.bash_login：用户个人配置
  • ～/.profile：用户个人配置

bash shell的数据流重定向

  • 标准输入（stdin）：代码为0，使用<或<<。<是将原本需要键盘输入的数据改由文件内容来替代，<<是代表结束输入的意思，例如，<< “eof”表示当由键盘输入eof时，该次输入就结束；
  • 标准输出（stdout）：代码为1，使用>或>>。>是以覆盖的方式将正确的数据输出到指定的文件或者设备上，>>是以追加的方式将正确的数据输出到指定的文件或者设备上；
  • 标准错误输出（stderr）：代码为2，使用2>或2>>。2>是以覆盖的方式将错误的数据输出到指定的文件或者设备上，2>>是以追加的方式将正确的数据输出到指定的文件或者设备上；
  • /dev/null 垃圾桶黑洞设备

bash shell多命令执行的判断依据

  • cmd1 ; cmd2：命令1和2之间没有相关性依据；

  • cmd1 && cmd2：命令1执行成功，命令2才会执行，命令1执行失败，命令2不执行；

  • cmd1 || cmd2：命令1执行成功，命令2不执行，命令1执行失败，命令2才会执行；

  • 命令回传码$?：命令执行成功则为0，命令执行失败则不为0。

Linux运维—磁盘配额（Quota）

Posted on 2022-12-16 | Post modified: 2023-04-13 | In linux

Quota功能需要文件系统的支持。

  • mount -o remount,usrquota,grpquota 文件系统名：文件系统手动加入quota支持，如果需要一直生效，则需要修改/etc/fstab文件；
  • quotacheck -avug 文件系统名：构建文件系统的quota配置文件,构建的配置文件会放到指定的文件系统下面；
  • quotaon -avug 文件系统名：启动指定文件系统的quota功能；
  • quotaoff -aug 文件系统名：关闭指定文件系统的quota功能；
  • edquota [-u username] [-g groupname]：修改指定用户或用户组的限值和宽限时间；
  • quota [-uvs username] [-gvs groupname]：查询指定用户或用户组的quota限额；
  • repquota -augvs：查看所有启用quota功能的文件系统的所有用户和用户组的quota限额；

quota限额设置的七个字段：

  • 文件系统名称：表示该quota限额是针对哪个文件系统的；
  • 已使用的block（磁盘容量）：这个数值是quota自己算出来的，不用修改；
  • soft：block的soft限制值；
  • hard：block的hard限制值；
  • 已使用的inode（文件数量）：这个数值是quota自己算出来的，不用修改；
  • soft：inode的soft限制值；
  • hard：inode的hard限制值；

Mysql高级—常见超时参数的含义

Posted on 2022-12-12 | Post modified: 2022-12-12 | In database

查询mysql中的各种超时参数

连接、网络类超时

列表如下：
- connect_timeout: 默认为10秒
- wait_timeout: 默认是28800秒，8小时
- interactive_timeout: 默认是28800秒，8小时
- net_read_timeout: 默认是30秒
- net_write_timeout: 默认是60秒

connect_timeout

在获取连接阶段起作用，

  • 获取MySQL连接是多次握手的结果，除了用户名和密码的匹配校验外，还有IP->HOST->DNS->IP验证，任何一步都可能因为网络问题导致线程阻塞。为了防止线程浪费在不必要的校验等待上，超过connect_timeout的连接请求将会被拒绝。

interactive_timeout和wait_timeout

在连接空闲阶段（sleep）起作用

  • 即使没有网络问题，也不能允许客户端一直占用连接。对于保持sleep状态超过了wait_timeout（或interactive_timeout，取决于client_interactive标志）的客户端，MySQL会主动断开连接。

net_read_timeout和net_write_timeout

在连接查询阶段（query）起作用

  • 客户端忙于计算或者存储数据，MySQL也选择了有条件的等待。在数据包的分发过程中，客户端可能来不及响应（发送、接收、或者处理数据包太慢）。

连接参数生效过程

在TCP三次握手的基础之上，简历MySQL通讯协议的连接，这个连接建立过程受connect_timeout参数控制
——————–TCP established——————–
MySQL Server(10.10.20.96)——->Client(10.10.20.51)
Client(10.10.20.51)——->MySQL Server(10.10.20.96)
MySQL Server(10.10.20.96)——->Client(10.10.20.51)
——————–established——————–

在MySQL通讯协议建立连接之后，此时客户端连接的超时受wait_timeout和interactive_timeout参数控制
建立连接后无交互：MySQL server —wait_timeout— Client
建立连接交互后：MySQL server —interactive_timeout— Client

在如果客户端有数据包传输，那么这个数据包的传输超时由net_read_timeout和net_write_timeout参数控制
——————-client与server端有数据传输时——————-
client —–>MySQL Server(net_read_timeout)
client <—–MySQL Server(net_write_timeout)

锁类超时

列表如下：
- innodb_lock_wait_timeout
- innodb_rollback_on_timeout
- lock_wait_timeout

innodb_lock_wait_timeout

innodb使用这个参数能够有效避免在资源有限的情况下产生太多的锁等待；指的是事务等待获取资源时等待的最长时间，超过这个时间还未分配到资源则会返回应用失败；参数的时间单位是秒，最小可设置为1s(一般不会设置得这么小)，最大可设置1073741824秒（34年，一条语句锁等待超过30分钟估计业务该有反馈了），默认安装时这个值是50s，超过这个时间会报 ERROR 1205 (HY000): Lock wait timeout exceeded; try restarting transaction

innodb_rollback_on_timeout

默认情况下innodb_lock_wait_timeout 超时后只是sql执行失败，整个事务并不回滚不提交，如需要事务在超时的时候回滚，则需设置，innodb_rollback_on_timeout =ON，默认为OFF，这个参数控制回滚时间

lock_wait_timeout

凡需要获取MDL锁的操作都受到这个超时参数的影响，不单单是DDL语句，包含在表上的DML、DDL操作，以及视图、存储过程、存储函数、lock table，flush table with read lock语句等。但不适用于隐式访问系统表的语句，如：grant和revoke等

复制类超时

列表如下：
- slave_net_timeout

slave_net_timeout

Slave判断主库是否挂掉的超时设置，在设定时间内依然没有获取到Master的回应就认为Master已经挂掉了，后续根据超时重连参数设置进行重连主库的操作。默认值：3600S

工作流传参

  • 本地参数
    作用于工作流的当前任务节点，在任务定义节点进行配置。

  • 全局参数
    作用于工作流的所有任务节点，在保存工作流的时候进行配置。

  • 内置参数

    • 基础内置参数

      | 变量名              | 声明方式               | 含义                                            |
|---------------------|------------------------|-------------------------------------------------|
| system.biz.date     | ${system.biz.date}     | 日常调度实例定时的定时时间前一天，格式为yyyyMMdd  |
| system.biz.curdate  | ${system.biz.curdate}  | 日常调度实例定时的定时时间，格式为yyyyMMdd       |
| system.biz.datetime | ${system.biz.datetime} | 日常调度实例定时的定时时间，格式为yyyyMMddHHmmss |
* 衍生内置参数
    - 使用add_months()函数，该函数用于加减月份，第一个入口参数为[yyyyMMdd]，表示返回时间的格式，第二个入口参数为月份偏移量，表示加减多少个月
        
    - 在自定义格式后直接“+/-”数字，单位为天
