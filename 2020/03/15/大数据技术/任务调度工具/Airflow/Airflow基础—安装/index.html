Airflow安装文档
准备工作，用户准备：
useradd airflow
useradd –d /home/airflow -m airflow
passwd airflow

etl_usr 用户，无需用户目录（已创建）
准备工作，权限相关准备：
• 将hive用户加入到supergroup组中（应急方案）
• etl_usr 在sentry下修改，将Server1 赋给etl_usr（应急方案）
准备工作，网络相关准备
• Rabbitmq的防火墙 15672
• Airflow 的防火墙 8081
• Mysql 的防火墙 3306
• Flower 的防火墙 5555
准备工作，组件分布：

|—|—|—|
|Webserver|bdputil01, bdputil02|监控门户|
|scheduler|bdputil01, bdputil02|调度器|
|Worker|bdpdata01, bdpdata02|脚本执行|
|Queuing service|bdpcm02, bdputil01, bdputil02|消息通讯|
|Flower|bdputil01|Worker监控|
|元数据节点|Mysql|bdpkylin01, bdpkylin02|ETL元数据|

组件名称 组件版本

MySQL Mysql-5.6.37
RabbitMQ Rabbitmq-3.3.5

准备工作，创建key.tab：
在bdputil01，bdputil02，bdpdata01，bdpdata02的/home/airflow目录下执行如下命令：
ktutil
addent -password -p etl_usr@SPD BBIZ.COM -k 1 -e rc4-hmac
addent -password -p etl_usr@SPDBBIZ.COM -k 1 -e aes256-cts
wkt etl_usr.keytab
quit

failover免密
在scheduler所在的节点上做airflow用户的免密操作，免密登录本机与其他机器。
配置airflow
在所有组件节点上配置环境变量
vi /etc/profile 在末尾添加上
export AIRFLOW_HOME=/home/airflow
PATH=$PATH:$AIRFLOW_HOME
export PATH
source /etc/profile
修改配置文件
修改所有组件节点的airflow配置文件
配置文件路径：/home/airflow/airflow.cfg

executor = CeleryExecutor

The Celery broker URL. Celery supports RabbitMQ, Redis and experimentally

a sqlalchemy database. Refer to the Celery documentation for more

information.

broker_url = amqp://guest:spdb1234@bdputil01:5673

Another key Celery setting

celery_result_backend = db+mysql://airflow:spdb1234@bdpkylin01:3306/airflow

Default queue that tasks get assigned to and that worker listen on.

default_queue = spdb

failover获取hostname

scheduler_nodes_in_cluster = bdputil01,bdputil02
airflow_scheduler_start_commant = nohup airflow scheduler >> ~/logs/scheduler.logs

FAILOVER配置
使用airflow用户在bdputil01和bdputil02上进行如下操作：
修改私钥名称
mv /home/airflow/.ssh/id_rsa /home/airflow/.ssh/id_rsa.bak
做好两台机器的免密登陆，将私钥(id_rsa)拷贝到/home/airflow目录下，修改名称为heartbeat-key
备份command_runner.py
mv /home/airflow/airflow-scheduler-failover-controller/scheduler_failover_controller/command_runner/command_runner.py /home/airflow/airflow-scheduler-failover-controller/scheduler_failover_controller/command_runner/command_runner.py.bak
将{}/command_runner.py复制到/home/airflow/airflow-scheduler-failover-controller/scheduler_failover_controller/command_runner目录下
完成上述步骤后，在/home/airflow/airflow-scheduler-failover-controller 执行如下命令进行验证
scheduler_failover_controller is_scheduler_running
管理Airflow
初始化airflow
在所有组件节点上，用root用户执行如下操作：
chown -R airflow:app /home/airflow
在其中一个节点AIRFLOW_HOME下执行：
su airflow
airflow initdb
初始化airflow-failover-controller
在scheduler所在节点上初始化
用airflow用户，airflow-scheduler-failover-controller目录下，执行如下操作：
scheduler-failover-controller init
启动web：
在webserver所在的节点以airflow用户在/home/airflow目录执行：
nohup airflow webserver -p 8081 >> /home/airflow/logs/webserver.logs &
启动failover：
在scheduler所在节点上以airflow用户在/home/airflow/airflow-scheduler-failover-controller执行：
nohup scheduler_failover_controller start >> /home/airflow/logs/scheduler.logs &
启动airflow worker：
在worker所在节点上以airflow用户在/home/airflow执行：
nohup airflow worker >> /home/airflow/logs/worker.logs &
