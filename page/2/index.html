Java基础—集合

Posted on 2020-03-11 In java

Arraylist 、LinkedList 区别

  • 是否保证线程安全： ArrayList 和 LinkedList 都是不同步的，也就是不保证线程安全。
  • 底层数据结构： Arraylist 底层使用的是 Object 数组；LinkedList 底层使用的是双向链表数据结构。
  • 插入和删除是否受元素位置的影响： ① ArrayList 采用数组存储，所以插入和删除元素的时间复杂度受元素位置的影响。 比如：执行add(E e)方法的时候， ArrayList 会默认在将指定的元素追加到此列表的末尾，这种情况时间复杂度就是O(1)。但是如果要在指定位置 i 插入和删除元素的话（add(int index, E element)）时间复杂度就为 O(n-i)。因为在进行上述操作的时候集合中第 i 和第 i 个元素之后的(n-i)个元素都要执行向后位/向前移一位的操作。 ② LinkedList 采用链表存储，所以对于add(E e)方法的插入，删除元素时间复杂度不受元素位置的影响，近似 O（1），如果是要在指定位置i插入和删除元素的话（(add(int index, E element)） 时间复杂度近似为o(n))因为需要先移动到指定位置再插入。

Java易错点总结

Posted on 2020-03-11 In java

JVM、JDK和JRE的区别

  • JVM是运行Java字节码的虚拟机。
  • JDK是运行已编译的Java程序的所有内容的集合，包含JVM、java类库、java命令和一些基础构件。
  • JDK是功能齐全的Java SDK，包含JRE，编译器和工具。

    Override、Overload的区别

  • 重写（Override）是子类对父类的允许访问的方法的实现过程进行重新编写,发生在子类中，方法名、参数列表必须相同，返回值范围小于等于父类，抛出的异常范围小于等于父类，访问修饰符范围大于等于父类。
  • 重载（Overload）是发生在同一个类中，方法名必须相同，参数类型不同、个数不同、顺序不同，方法返回值和访问修饰符可以不同。

hadoop核心技术（二）—hdfs元数据管理剖析

Posted on 2019-05-14 Edited on 2020-03-11 In bigdata

##概述
Hdfs的元数据存储在namenode的内存中，为了防止namenode宕机导致所有的元数据丢失，需要定期备份内存中的数据到硬盘，但如果元数据内容过大，则在dump时会耗费很长时间并且会影响hdfs的性能，因此出现了secondnamenode的角色。二者配合进行元数据管理。
##元数据管理
元数据管理流程图：

流程：

  1. Client端跟nn（namenode）交互会造成nn元数据的更新，同时基于WAL思想，所有涉及到更新元数据的操作都会写入编辑日志中（edits-in-progress文件）。
  2. snn（secondarynamenode）的进程会定期向nn的进程发起合并edits和fsimage的请求。
    1. nn根据时间和edits文件大小进行判断是否需要合并
    2. 如果需要进行合并则先触发edits文件（图中的edits-in-progress-0009-000X）的回滚操作，生成需要合并的edits文件（edits-0009-0012）和新的edits文件（edits-in-progress-0013-000X）
    3. nn将需要合并的edits文件（edits-0009-0012）发送到snn
    4. snn将上一次dump的fsimage（fsimage-0008）加载到内存中，然后将收到的edits文件（edits-0009-0012）依次加载到内存中
    5. snn将内存中的元数据dump成新的fsimage（fsimage-0012）写入磁盘
    6. Snn将新的fsimage发送给nn，nn接收成功后清除旧的fsimage和edits文件

Posted on 2019-05-14 Edited on 2020-03-11 In bigdata

hdfs写数据文件流程

hdfs写入数据流程图如下：

流程：

  1. client连接namenode存数据文件。
  2. namenode记录一条数据位置信息（元数据，一个元数据大概150b），告诉client存哪里。
  3. client用hdfs的api将数据块（默认128M）存储到datanode上（选择最近的一个datanode建立网络连接）。
  4. datanode将数据水平备份（客户端决定该文件的副本数量）。并且备份完成反馈给client。
  5. client通知namenode存储块完毕。
  6. namenode将元数据同步到内存中。
  7. 另一块重复上面的过程。
