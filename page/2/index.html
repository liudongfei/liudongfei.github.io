Linux运维—磁盘分区

Posted on 2022-04-11 | Post modified: 2023-04-13 | In linux

磁盘分区操作

  • 查看磁盘分区：fdisk -l
  • 添加磁盘分区：fdisk 磁盘设备名称
  • 删除磁盘分区：fdisk 磁盘设备名称
  • 格式化磁盘分区：mkfs
  • 磁盘分区检查：fsck
  • 磁盘分区挂载：mount 分区名称 挂载点
  • 磁盘分区卸载：unmount 挂载点
  • 设置开机挂载：编辑/etc/fstab

构建内存交换空间（swap）

  • 添加磁盘分区：fdisk 磁盘设备名称
  • 格式化磁盘分区为swap格式：mkswap 磁盘分区名称
  • 查看原本swap使用量：free
  • 开启新的swap：swapon 磁盘分区名称
  • 查看当前所有的swap设备：swapon -s

常见的磁盘阵列

RAID的常见等级：

  • RAID-0：性能最佳，安全性最差，任意一块磁盘损坏都会导致文件丢失。总容量等于所有磁盘的容量之和；
  • RAID-1：最大的优点就是数据的备份，总容量等于所有磁盘的容量之和的一半；
  • RAID-5：至少需要三块磁盘才可以组成这种类型的磁盘阵列，通过使用同位检查码，确保任何一块磁盘损坏都会通过其他磁盘的检查码来恢复数据。总容量等于整体磁盘的数量减去一块；这种类型的磁盘阵列只能允许一块磁盘损坏；

Linux的逻辑卷管理器（Logical Volume Manager）

LVM的重点在于可以弹性调整文件系统的容量。

  • PV：物理卷，实际的物理分区，是LVM的最底层；
  • PE：物理扩展块，类似文件系统的block，默认为4M；
  • VG：卷用户组，由PE组成了VG，最大可以有256G的容量；
  • LV：逻辑卷，由VG切分出来成为LV。

Linux运维—文件系统

Posted on 2022-03-13 | Post modified: 2023-04-13 | In linux

文件系统Ext2

文件系统Ext2的介绍

操作系统Ext2中的文件除了文件除了文件的实际内容外，通常含有更多的属性，如文件权限（rwx）与文件属性（所有者、群组、时间属性）等。
文件系统通常会将这两部分不同的数据存放在不同的块，权限与属性存放到inode中，至于实际数据则放置到data block块中。此外还有一个super block记录整个文件系统的使用数据，包括inode与block的总量、使用量、剩余量等。
总体上，文件系统包含如下三部分：

  • super block：记录文件系统的整体信息，包括indoe与block的总量、使用量、剩余量，以及文件系统的格式与相关信息等；
  • inode：记录文件的属性，一个文件占用一个inode，，同时记录此文件的数据所在block号码；
  • block：记录文件的实际内容，若文件过大，会占用多个block。block的大小有1kb，2kb，4kb三种。

###Ext2和FAT文件系统的区别
在文件系统Ext2中，每一个文件都会占用一个inode，inode里面则有文件存放的block号码。系统读取文件时会先找到inode，然后分析inode记录的权限与用户是否符合，若符合才能实际读取block的数据内容。这种方法被称为索引式文件系统。
U盘的文件系统一般是FAT，这种文件系统没有inode在，因此FAT没有办法将这个文件的所有block块一次性都读出来，因此下一个block块的号码存放在前一个block块中，因此如果一个文件数据写入的block块过于分散，则磁盘读取到这个文件的效率就会过低。所谓的“碎片整理”就是文件写入的block块过于分散了，要将同一个文件的block汇总到一起。

Ext2的block属性

  • block的大小与数量在格式化完成就不能够再改变了；
  • 每个block内最多只能够放置一个文件的数据，如果文件大于block的大小，则一个文件会占用多个block块。

Ext2的inode属性

  • 每个inode的大小均固定为128btyes；
  • 每个文件都只会占用一个inode，每个inode记录block号码的区域分为12个直接、1个间接、一个双间接、一个三间接。

Ext2的Superblock属性

  • block与inode的总量；

  • 未使用与已使用的inode/block数量；

  • block与inode的大小；

  • block对照表；

  • inode对照表；

  • 文件系统描述。

    文件系统的查看操作

  • df：列出文件系统的整体磁盘使用量

  • du：评估文件系统的磁盘使用量，常用于评估目录所占的容量。

Linux运维—目录与文件权限

Posted on 2022-03-08 | Post modified: 2023-04-13 | In linux

目录与文件权限的意义

Linux系统内文件的三种身份：所有者、用户组和其他人。每种身份都有三种权限（r、w、x），用数字4、2、1表示。
对于文件：

  • r：可读取此文件的实际内容。
  • w：可以编辑、新增或者修改该文件的内容（但是不含删除该文件）。
  • x：该文件具有可以被系统执行的权限。

总而言之，对于文件的r、w、x权限来说，主要都是针对该文件的内容而言，与文件的存在与否没有关系。

对于目录：

  • r：表示具有读取目录结构列表的权限。可以利用ls这个命令将这个目录的内容列表显示出来。
  • w：表示具有更改该目录结构列表的权限。主要包含：新建新的文件与目录；删除已经存在的文件与目录（不论该文件的权限为何）；将以存在的文件或目录进行重命名；转移该目录内的文件、目录位置。
  • x：表示进入该目录的权限。可以使用cd这个命令切换到这个目录中。

总之，要开放目录给任何人浏览时，至少要给与x和r的权限，如果只有r权限就只能查询到目录下的文件名称，并不能切换到这个目录中。
默认情况下，用户创建文件是没有可执行（x）权限的，只有r、w两项，也就是最大为666。用户新建目录，则x与是否可进入此目录有关，因此默认所有权限都开放，即为777。
用户新建文件或目录时默认的权限由umask命令的后面三个数字控制，umask的分数指的是“该默认值需要减掉的权限”。
root用户的umask一般是0022，后三位即022，所有者拥有全部权限，用户组去除写权限，其他去除写权限。在这种情况下，用户创建的文件权限默认为644，目录的默认权限为755。普通用户的umask是0002。

Linux运维—Linux的由来

Posted on 2022-02-27 | Post modified: 2022-03-07 | In linux

Linux之前UNIX的历史

  • 1969年以前，输入设备只有卡片阅读机，输出设备只有打印机，用户无法与操作系统进行互动。后来设备更新和操作系统的改良，可以使用键盘进行输入。此时，如果多人使用一台主机，则需要等待。后来麻省理工学院开发了分时操作系统，可以使大型主机通过提供多个终端连接进入主机，利用主机资源进行运算工作，但是终端只具备输入/输出的功能，本身不具有任何运算或软件安装的能力，同时比较先进的主机也只能链接30台左右的终端机。后来为了增加大型主机的功能，在1965年前后由贝尔实验室、麻省理工学院及通用电气公司共同发起了Multics项目，为了让大型主机同时提供300台以上的终端连接机使用的目的。在1969年前后，由于项目进度落后，资金短缺，贝尔实验室推出了该项目。
  • 1969年，原本参与Multics项目的成员Thompson，为了自己需要，以汇编语言(Assemler)写出来一组内核程序，同时包括一些内核工具程序及一个小的文件系统。该系统就是UNIX的原型。
  • 1973年，UNIX正式诞生。由于Thompson写的那个操作系统太好用了，在贝尔实验室内部广为流传，但是，由于是由汇编语言编写的，和计算机架构的兼容性太差，因此Thompson和Ritchie合作用C语言重新改写和编译，最后发行出了UNIX的正式版本。
  • 1977年，重要的UNIX分支—BSD诞生。贝尔实验室属于AT&T公司，对于UNIX采取较为开放的态度，在1973年后UNIX便得以与学术界合作开发，最重要的就是与加州柏克莱（Berkeley）大学合作。柏克莱大学的BillJoy在取得UNIX的源码后，着手修改为舍和自己机器的版本，同时增加了很多工具软件与编译程序，最终将它命名为Berkeley Software Dsitribution（BSD）。这个BSD是UNIX很重要的一个分支，Bill Joy也是Sun这家公司的创办者。Sun公司即是以BSD开发的内核进行自己的商业UNIX版本的开发的。
  • 1979年，由于UNIX的高度可移植性与强大的性能，加上当时并没有版权的纠纷，所以很多公司开始了UNIX操作系统的开发，例如AT&T的System V、IBM的AIX等。而在早期每家生产计算机硬件的公司还没有“协议”的概念，所以每一家计算机公司生产的硬件自然就不相同，因此需要为自己的计算机硬件开发合适的UNIX系统，而且仅能与自身的硬件支持，这种情况在1979年AT&T推出System V第七版UNIX后有所改善，这个版本的System V支持x86架构的个人计算机。由于商业原因，AT&T在第七版的UNIX中，特别提到“不可对学生提供源码”的严格限制
  • 1984年，Richard Mathew Stallman（史托曼）发起了GNU项目，对于当今自由软件风潮具有不可磨灭的地位。这个项目是创建一个自由、开放的UNIX操作系统(Free UNIX)，著名的GNU C Compiler（GCC）、Emacs以及常用的C函数库、bash shell等都是他完成的。为了避免GNU开发的自由软件被其他所利用成为专利软件，他与律师草拟了有名的通用公共许可证（GPL）。
  • 1991年，芬兰的赫尔辛基大学的Linus Torvalds以bash，gcc等工具写了一个小小的内核程序，这个内核程序可以在Intel的386机器上运行。

Linux的内核版本

  • 主、次版本为奇数：开发中版本。主要在测试和开发新功能。
  • 主、次版本为偶数：稳定版本

Linux的Distributions

  • Linux的Distributions包括“Kernel + Software + Tools”
  • 一般Distribution主要分为两大系统，一种使用RPM方式安装软件的系统，如Red Hat，Fedora，SuSE等；一种则是使用Debian的dpkg方式，如Debian，Ubuntu，B2D等。
  • 如何选择Distribution？如果装在个人计算机上，建议使用团队版Fedora，Ubuntu，OpenSuSE等。如果用在服务器上建议使用商业版本，如Red Hat，SuSE。

Linux运维—计算机概论

常用的计算单位

  • 大小单位：0/1的单位称为bit。存储数据时每份简单的数据都会使用到8个bit的大小来记录，称之为Byte；1Byte=8bit。
进制位 K M G T P
二进制 1024 1024K 1024M 1024G 1024T
十进制 1000 1000K 1000M 1000G 1000T
  • 速度单位：Hz是秒分之一。网络传输方面往往采用bit为单位，如网络常使用Mbit/s，即每秒多少Mbit，如果转成文件容量Byte时，其实需要除以8。

CPU

  • CPU的种类：按照指令集的分类，分为精简指令集（RISC）与复杂指令集（CISC）。
  • CPU的频率：频率即CPU每秒钟可以进行的工作次数。如频率为3.0GHz，表示CPU再一秒钟之内可以进行3.0 * 10的9次方次工作。
  • CPU的外频与倍频：外频指的是CPU与外部组件（内存）进行数据传输/运算时的速度，倍频则是CPU内部用来加速工作性能的一个倍数，两者相乘是CPU的频率。
  • CPU的32位与64位：内存与CPU的通信速度考的是外部频率，那么每次工作可以传送的数据量有多大就是总线的功能。一般主板芯片组分为北桥和南桥，北桥的总线成为系统总线，因为内存传输的主要信道，所以速度较快；南桥就是所谓的输入输出总线。北桥所支持的频率称为前端总线速度（FSB），而每次传送的位数则是总线宽度，总线频宽=FSB * 总线宽度，常见的总线宽度有32/64位（bit）。与总线宽度相似，CPU每次能够处理的数据量称为字组大小，字组大小依据CPU的设计而有32位与64位。现在所称的计算机是32位或者64位主要是依据CPU解析的字组大小而来。

内存

  • DRAM：动态随机访问内存。通电才记录与使用，断电数据消失。就是常用的与CPU交互的主要数据内存。
  • SRAM：静态随机访问内存。用于CPU高速缓存，一般集成到CPU内
  • ROM：只读存储器。通常用于BIOS程序写入。

显卡

显卡又称为VGA，显卡上会有一个内存容量，这个显卡内存将会影响到最终的屏幕分辨率和色彩深度。除了内存外，由于3D动画的流行，因此显卡的运算能力也越来越重要。一般会在显卡嵌入3D加速的芯片，这就是GPU的由来。

硬盘与存储设备

  • 硬盘：由盘片、机械手臂、磁头与主轴马达所组成。盘片上由圆心以放射的方式分隔出磁盘的最小存储单位—扇区，每个扇区的最小单位为512bytes，这个值不可变。扇区组成一个圆就是一个磁道，在所有的盘片上同一个磁道可以组成一个柱面，柱面也是我们分隔硬盘的最小单位。
  • 计算硬盘的存储容量：磁头的数量*每次磁头负责的柱面数量 * 每个柱面所含有的扇区数量 * 扇区的容量

文字编码系统

  • 常用的英文编码表为ASCII系统，每个字符占1B的容量，因此总共会有2的8次方256个字符。至于中文编码最常用的就是gb2312系统，每个字符占2B的容量，因此总共会有2的16次方65536个字符。
  • 由于中文编码对某些些数据库系统来说会出现误判和乱码问题，同时其他非英语系国家同样出现了这些问题，为了解决这个问题，由国际组织ISO/IEC制定了所谓的Unicode编码系统，即我们常称呼的UTF8。

操作系统

  • 操作系统的组成：操作系统由下到上分别由硬件、内核、系统调用和应用程序4个层面构成。
  • 硬件层：计算机系统主要由硬件组成。包含上面的CPU、内存、显卡等
  • 内核层：主要在于管理计算机的所有活动，驱动硬件，提供合理的计算机系统资源分配，因此硬件不同，内核就要进行修改。内核的主要功能有：系统调用接口；程序管理；内存管理；文件系统管理；设备驱动。
  • 系统调用层：为了保护内核，同时为了更方面程序员比较容易开发软件，会提供一整组的开发接口，这就是系统调用层。

Mysql高级—常见坑与解决方案

Posted on 2022-01-06 | Post modified: 2022-12-12 | In database

  1. 问题现象：日志中出现World-writable config file ‘/etc/mysql/my.cnf’ is ignored。
    问题本质：mysql配置文件不生效。
    解决方法：mysql配置文件权限过大777，任何用户都可修改。

    使用`chmod 644 /etc/mysql/my.cnf`修改文件权限即可。

  2. 问题现象：mysql 1346错误，更新表记录报 “Field doesn’t have a default value” 错误。
    问题本质：mysql5.7版本以后默认增加了mysql的严格模式，在更新时会默认加入所有字段，严格模式定义即

    sql-mode="STRICT_TRANS_TABLES,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION"中的STRICT_TRANS_TABLES。

    解决方法：修改sql-mode，修改为

    sql-mode="NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION"
