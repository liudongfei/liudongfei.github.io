Graphframe基础—使用手册

Posted on 2020-03-15 | Post modified: 2020-03-15 | In util

用户使用手册
给出了关于GraphFrames的基本查询，模式查询和一些最基本的图算法，包含python和scala编码
创建GraphFrames
用户可以从顶点和边的DataFrame中创建
顶点的DataFrame应该包含一个列名为“id”的特殊列，这列指定了整个图中每个顶点的唯一ID
边的DataFrame应该包含两个列名为“src“（边的起点ID）和”dst“（边的终点ID）
两种DataFrame可以包含任意的其他列，这些列代表了顶点和边的属性。
用户也可以从一个包含边信息的DataFrame中创建，图中的顶点由边的起点和终点确定。
下面的例子是如何从顶点和边的DataFrame中创建GraphFrame
Scala代码：
import org.graphframes.GraphFrame
// Vertex DataFrame
val v = sqlContext.createDataFrame(List(
(“a”, “Alice”, 34),
(“b”, “Bob”, 36),
(“c”, “Charlie”, 30),
(“d”, “David”, 29),
(“e”, “Esther”, 32),
(“f”, “Fanny”, 36),
(“g”, “Gabby”, 60)
)).toDF(“id”, “name”, “age”)
// Edge DataFrame
val e = sqlContext.createDataFrame(List(
(“a”, “b”, “friend”),
(“b”, “c”, “follow”),
(“c”, “b”, “follow”),
(“f”, “c”, “follow”),
(“e”, “f”, “follow”),
(“e”, “d”, “friend”),
(“d”, “a”, “friend”),
(“a”, “e”, “friend”)
)).toDF(“src”, “dst”, “relationship”)
// Create a GraphFrame
val g = GraphFrame(v, e)
上面创建的GraohFrame可以在下面的包中找到

模式查询
表示在一个图中搜寻指定的结构样式
GraphFrame的模式查询使用一种简单Domain-Specific Language (DSL)表示结构查询。 例如，raph.find(“(a)-[e]->(b); (b)-[e2]->(a)”)查询一对双向互通的顶点，返回一个DataFrame，包含图中所有这种结构，列名就是表达式中的元素名称，在这个例子中，返回的列名是”a, b, e, e2.”
DSL表达式： 模式中最基本的单元是边，例如”(a)-[e]->(b)”表示一个从顶点a到顶点b的边e，顶点使用圆括号（a）表示，边使用方括号[e]表示。 一个表达式可以由一系列的边表示，用分号隔开,例如”(a)-[e]->(b); (b)-[e2]->(c)”表示a到b到c的两条边 在一个表达式中，边和顶点的名称是可以指定的，例如”(a)-[e]->(b); (b)-[e2]->(c)”表示边e的终点和边e2的起点’b’是相同的
这样命名有两个目的：
可以区分边之间的元素，例如例如”(a)-[e]->(b); (b)-[e2]->(c)”表示边e的终点和边e2的起点’b’是相同的
用于结果中的一列的列名
要知道不同的表名可能表示相同的顶点，例如，中的’a’和’c’可能表示相同的顶点
省略不需要的顶点和边的名称也是可以的，例如表示’a，b’之间的边，但是不用命名，结果中不会显示该列，被称为匿名的边和列
表达式中的边可以使用否定来表示这条边不会在图中出现，例如查找’a’到’b’但是’b’到不了’a’的边
规定：
不允许出现不包含任何元素名称的模式表达式出现，例如”()-[]->()”、”!()-[]->()”
被命名的边中不允许包含被排除的顶点名称，例如”!(a)-[ab]->(b)”是非法的，”!(a)-[]->(b)”是合法的
更多复杂的查询，例如对顶点或者边属性的操作可以使用表达式对结果的DataFrame进行过滤
Scala代码：
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
// Search for pairs of vertices with edges in both directions between them.
val motifs: GraphFrame = g.find(“(a)-[e]->(b); (b)-[e2]->(a)”)
motifs.show()
// More complex queries can be expressed by applying filters.
motifs.filter(“b.age > 30”).show()

许多的模式查询都是非常简单的，就如上面的出现的一样，下面的这些携带状态的例子更加复杂，这样的查询可以结合GraphFrame的模式查找和结果过滤，对结果中的列使用表达式操作进行过滤。

例如，
For example, suppose one wishes to identify a chain of 4 vertices with some property defined
by a sequence of functions. That is, among chains of 4 vertices a->b->c->d, identify the subset
of chains matching this complex filter:

  • Initialize state on path.
  • Update state based on vertex a.
  • Update state based on vertex b.
  • Etc. for c and d.
  • If final state matches some condition, then the chain is accepted by the filter.

The below code snippets demonstrate this process, where we identify chains of 4 vertices
such that at least 2 of the 3 edges are “friend” relationships.
In this example, the state is the current count of “friend” edges; in general, it could be any
DataFrame Column.
Scala代码：
import org.apache.spark.sql.Column
import org.apache.spark.sql.functions.{col, when}
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
// Find chains of 4 vertices.
val chain4 = g.find(“(a)-[ab]->(b); (b)-[bc]->(c); (c)-[cd]->(d)”)
// Query on sequence, with state (cnt)
// (a) Define method for updating state given the next element of the motif.
def sumFriends(cnt: Column, relationship: Column): Column = {
when(relationship === “friend”, cnt + 1).otherwise(cnt)
}
// (b) Use sequence operation to apply method to sequence of elements in motif.
// In this case, the elements are the 3 edges.
val condition = { Seq(“ab”, “bc”, “cd”)
.foldLeft(lit(0))((cnt, e) => sumFriends(cnt, col(e)(“relationship”))) }
// (c) Apply filter to DataFrame.
val chainWith2Friends2 = chain4.where(condition >= 2)
chainWith2Friends2.show()
Python代码：
from pyspark.sql.functions import col, lit, when
from pyspark.sql.types import IntegerType
from graphframes.examples import Graphs
g = Graphs(sqlContext).friends() # Get example graph
chain4 = g.find(“(a)-[ab]->(b); (b)-[bc]->(c); (c)-[cd]->(d)”)

Query on sequence, with state (cnt)

(a) Define method for updating state given the next element of the motif.

sumFriends = lambda cnt,relationship: when(relationship == “friend”, cnt+1).otherwise(cnt)

(b) Use sequence operation to apply method to sequence of elements in motif.

In this case, the elements are the 3 edges.

condition = reduce(lambda cnt,e: sumFriends(cnt, col(e).relationship), [“ab”, “bc”, “cd”], lit(0))

(c) Apply filter to DataFrame.

chainWith2Friends2 = chain4.where(condition >= 2)
chainWith2Friends2.show()
上面这个例子是查找一个固定长度的链，显然，想要查询变量定义的这种模式，用户需要查询每一条可能的路线，然而上面的固定长度查询模式允许用户重复使用相同的代码，只需要更新模式识别中的元素 如（’ab’，’bc’，’cd’）
子图
在GraphX中，’subfraph‘方法接受一个边的triplet，允许用户根据triplet和顶点选择一个子图
GraphFrames提供了一个更强大的方式，基于模式识别和DataFrame筛选结合，来选择子图，我们提供了三个辅助的方法来辅助子图选择，filterVertices(condition), filterEdges(condition), and dropIsolatedVertices().
简单的子图，基于顶点和边过滤
下面的列子展示了如何基于边和顶点过滤选择子图
Scala代码：（0.6.0版本新增的API）
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends

// Select subgraph of users older than 30, and relationships of type “friend”.
// Drop isolated vertices (users) which are not contained in any edges (relationships).
val g1 = g.filterVertices(“age > 30”).filterEdges(“relationship = ‘friend’”).dropIsolatedVertices()
Python代码：

import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
// Select subgraph based on edges “e” of type “follow”

val paths = { g.find(“(a)-[e]->(b)”)
.filter(“e.relationship = ‘follow’”)
.filter(“a.age < b.age”) }

// val e2 = paths.select(“e.*”)
// Construct the subgraph

BFS算法
BFS算法查找从一个或者多个顶点到另一个或者多个顶点的最短路径，起始点和终点在表达式中被指定。See Wikipedia on BFS for more background.
下面的代码段使用bfs算法查找名称为Esther的顶点到年龄小于32的一系列顶点的最短路径
For API details, refer to the API docs.
Scala代码：
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
// Search from “Esther” for users of age < 32.
val paths = g.bfs.fromExpr(“name = ‘Esther’”).toExpr(“age < 32”).run()
paths.show()
// Specify edge filters or max path lengths.
{ g.bfs.fromExpr(“name = ‘Esther’”).toExpr(“age < 32”)
.edgeFilter(“relationship != ‘friend’”)
.maxPathLength(3).run() }

For API details, refer to the API docs.

标签传播算法

Label Propagation Algorithm (LPA)

在网络中运行一个静态的LPA算法去探测社交关系
Run static Label Propagation Algorithm for detecting communities in networks.
网络中的每个节点初始被赋予自己的社交圈，在每一步中，节点发送各自的社交圈给所有的相邻节点，更新他们的社交圈
Each node in the network is initially assigned to its own community. At every superstep, nodes
send their community affiliation to all neighbors and update their state to the mode community
affiliation of incoming messages.
LPA是一个标准的图社交探测算法。它是非常廉价的由于（1）收敛性是不保证的（2）可以以无关紧要的结果结束
LPA is a standard community detection algorithm for graphs. It is very inexpensive
computationally, although (1) convergence is not guaranteed and (2) one can end up with
trivial solutions (all nodes are identified into a single community).
See Wikipedia for background.

For API details, refer to the API docs.
Scala代码
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
val result = g.labelPropagation.maxIter(5).run()
result.select(“id”, “label”).show()
For API details, refer to the API docs.

PageRank
有两种实现方法
There are two implementations of PageRank.
第一种使用org.apache.spark.graphx.graph接口的aggregateMessage方法运行固定次数的迭代来实现，可以设置迭代次数
第二种实现使用接口org.apache.spark.graphx.Pregel运行直到收敛，可以设置容忍度

  • The first one uses the org.apache.spark.graphx.graph interface with aggregateMessages and runs
    PageRank for a fixed number of iterations. This can be executed by setting maxIter.
  • The second implementation uses the org.apache.spark.graphx.Pregel interface and runs PageRank until
    convergence and this can be run by setting tol.
    两种实现都支持自定义或非自定义的PageRank
    Both implementations support non-personalized and personalized PageRank, where setting a sourceId
    personalizes the results for that vertex.

See Wikipedia for background.
For API details, refer to the API docs.
Scala代码：
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
// Run PageRank until convergence to tolerance “tol”.
val results = g.pageRank.resetProbability(0.15).tol(0.01).run()
// Display resulting pageranks and final edge weights
// Note that the displayed pagerank may be truncated, e.g., missing the E notation.
// In Spark 1.5+, you can use show(truncate=false) to avoid truncation.
results.vertices.select(“id”, “pagerank”).show()
results.edges.select(“src”, “dst”, “weight”).show()
// Run PageRank for a fixed number of iterations.
val results2 = g.pageRank.resetProbability(0.15).maxIter(10).run()
// Run PageRank personalized for vertex “a”
val results3 = g.pageRank.resetProbability(0.15).maxIter(10).sourceId(“a”).run()
// Run PageRank personalized for vertex [“a”, “b”, “c”, “d”] in parallel
val results3 = g.parallelPersonalizedPageRank.resetProbability(0.15).maxIter(10).sourceIds(Array(“a”, “b”, “c”, “d”)).run()
For API details, refer to the API docs.

保存和加载GraphFrames

Saving and loading GraphFrames

由于GraphFrames是DataFrame创建的，自然就支持从相同的数据源保存和加载
Since GraphFrames are built around DataFrames, they automatically support saving and loading
to and from the same set of datasources.
Refer to the Spark SQL User Guide on datasources
for more details.
下面的例子展示如何保存和加载一个图
The below example shows how to save and then load a graph.
Scala代码：
import org.graphframes.{examples,GraphFrame}
val g: GraphFrame = examples.Graphs.friends // get example graph
// Save vertices and edges as Parquet to some location.
g.vertices.write.parquet(“hdfs://myLocation/vertices”)
g.edges.write.parquet(“hdfs://myLocation/edges”)
// Load the vertices and edges back.
val sameV = sqlContext.read.parquet(“hdfs://myLocation/vertices”)
val sameE = sqlContext.read.parquet(“hdfs://myLocation/edges”)
// Create an identical GraphFrame.
val sameG = GraphFrame(sameV, sameE)

GraphX to GraphFrame

GraphFrame提供了两种转换方法。第一种方法是将GrphX的图中的顶点和边的RDD转换成DataFrame，然后穿件GraphFrame
GraphFrame provides two conversions methods. The first takes any GraphX graph and converts
the vertex and edge RDDs into DataFrames using schema inference. Those DataFrames
are then used to create a GraphFrame.
第二种转换方案更加复杂
The second conversion method is more complex and is useful for users with existing GraphX code.
Its main purpose is to support workflows of the following form: (1) convert a GraphFrame to GraphX,
(2) run GraphX code to augment the GraphX graph with new vertex or edge attributes, and
(3) merge the new attributes back into the original GraphFrame.

For example, given:

  • GraphFrame originalGraph
  • GraphX Graph[String, Int] graph with a String vertex attribute we want to call “category”
    and an Int edge attribute we want to call “count”

We can call fromGraphX(originalGraph, graph, Seq("category"), Seq("count")) to produce
a new GraphFrame. The new GraphFrame will be an augmented version of originalGraph,
with new GraphFrame.vertices column “category” and new GraphFrame.edges column “count” added.

架构介绍
docker采用的是c/s架构，客户端向服务端发送请求，服务器负责构建、运行和分发容器。客户端和服务器可以运行在同一个host上，客户端也可以通过socket或restapi与服务器通信
组件介绍
• Docker Client：docker服务端的远程控制器，最常用的docker客户端是docker命令，也可以通过restapi与服务器通信
• Docker Server：docker doemoon是服务器组件，以linux后台服务的方式运行在docker host上，负责创建、运行、监控容器，构建、存储镜像
• Image：可以将docker镜像看做只读模版，通过它可以创建docker同期，例如某个镜像可能包含一个Linux操作系统、一个Apache HTTP Server，以及用户开发的Web应用。惊喜那个可以有多种生成方法，可以从无到有开始创建镜像，也可以下载并使用别人创建好的现成镜像，还可以在现有镜像的基础上创建新的镜像。我们可以将镜像的内容和创建的步骤描述在一个文本文件中，这个文件被称作dockerfile。通过执行docker build命令可以创建出镜像。
• Container：docker容器就是docker镜像的运行实例，用户可以通过CLI或者API启动、停止、移动或者删除容器，可以这么任务，镜像是软件生命周期的构建和打包阶段，而容器则是启动和运行阶段
• Registry：registry是存放docker镜像的仓库，registry分私有和公有两种。docekrhub是默认的registry，由docker公司维护。出于对速度和安全的考虑，用户可以创建自己的registry。dockerpull命令可以从registry下载镜像，docker run命令是先下载（如果本地没有），然后在启动容器。
• Volume：将物理机的文件夹挂载到容器的内部
• Dockerfile：用于创建镜像的脚本

Quartz概要介绍
Quartz是OpenSymphony开源组织在Job scheduling领域又一个开源项目，是完全由java开发的一个开源的任务日程管理系统，“任务进度管理器”就是一个在预先确定（被纳入日程）的时间到达时，负责执行（或者通知）其他软件组件的系统。
Quartz用一个小Java库发布文件（.jar文件），这个库文件包含了所有Quartz核心功能。这些功能的主要接口(API)是Scheduler接口。它提供了简单的操作，例如：将任务纳入日程或者从日程中取消，开始/停止/暂停日程进度。
Quartz定时器的种类
Quartz 中五种类型的 Trigger：
⁃ SimpleTrigger
⁃ CronTirgger
⁃ DateIntervalTrigger
⁃ NthIncludedDayTrigger
⁃ Calendar
Quartz的元数据存储方式

类型 优点 缺点
RAMJobStore 不要外部数据库，配置容易，运行速度快 调度信息存储在JVM内存，当应用程序停止运行时，所有调度信息将被丢失。另外因为存储到JVM内存里面，所以可以存储多少个Job和Trigger将会受到限制
JDBCJobStore 支持集群；所有任务信息都会保存到数据库中，可以控制事务；应用服务器关闭或者重启，任务信息都不会丢失，可以恢复因服务器关闭或者重启而导致执行失败的任务 运行速度的快慢取决与连接数据库的快慢

Quartz的核心类
QuartzSchedulerThread：负责执行向QuartzScheduler注册的触发Trigger的工作的线程。
ThreadPool：Scheduler使用一个线程池作为任务运行的基础设施，任务通过共享线程池中的线程提供运行效率。
QuartzSchedulerResources：包含创建QuartzScheduler实例所需的所有资源（JobStore，ThreadPool等）。
SchedulerFactory ：提供用于获取调度程序实例的客户端可用句柄的机制。
JobStore： 通过类实现的接口，这些类要为org.quartz.core.QuartzScheduler的使用提供一个org.quartz.Job和org.quartz.Trigger存储机制。作业和触发器的存储应该以其名称和组的组合为唯一性。
QuartzScheduler ：这是Quartz的核心，它是org.quartz.Scheduler接口的间接实现，包含调度org.quartz.Jobs，注册org.quartz.JobListener实例等的方法。
Scheduler ：这是Quartz Scheduler的主要接口，代表一个独立运行容器。调度程序维护JobDetails和触发器的注册表。 一旦注册，调度程序负责执行作业，当他们的相关联的触发器触发（当他们的预定时间到达时）。
Trigger ：具有所有触发器通用属性的基本接口，描述了job执行的时间出发规则。 - 使用TriggerBuilder实例化实际触发器。
JobDetail ：传递给定作业实例的详细信息属性。 JobDetails将使用JobBuilder创建/定义。
Job：要由表示要执行的“作业”的类实现的接口。只有一个方法 void execute(jobExecutionContext context)
(jobExecutionContext 提供调度上下文各种信息，运行时数据保存在jobDataMap中)
Job有个子接口StatefulJob ,代表有状态任务。有状态任务不可并发，前次任务没有执行完，后面任务处于阻塞等到。
