Spring核心—IOC和DI

Posted on 2020-03-14 | Post modified: 2020-03-16 | In java

IoC：Inverse of Control（控制反转）

  • 读作“反转控制”，更好理解，不是什么技术，而是一种设计思想，就是将原本在程序中手动创建对象的控制权，交由Spring框架来管理。
  • 正控：若要使用某个对象，需要自己去负责对象的创建
  • 反控：若要使用某个对象，只需要从Spring容器中获取需要使用的对象，不关心对象的创建过程，也就是把创建对象的控制权反转给了Spring框架
  • 好莱坞法则： Don’t call me ,I’ll call you

Spring基础—概述

Posted on 2020-03-14 | Post modified: 2020-03-16 | In java

什么是Spring

Spring框架是Java应用最广的框架，它的成功来源于理念，而不是技术本身，它的理念包括IoC (Inversion of Control，控制反转) 和 AOP(Aspect Oriented Programming，面向切面编程)。

Spring 是一个轻量级的 DI / IoC 和 AOP 容器的开源框架，来源于 Rod Johnson 在其著作 《Expert one on one J2EE design and development》 中阐述的部分理念和原型衍生而来。Spring提倡以“最少侵入” 的方式来管理应用中的代码，这意味着我们可以随时安装或者卸载 Spring。

  • 适用范围：任何 Java 应用
  • Spring 的根本使命：简化 Java 开发

Mysql基础—安装mysql

Posted on 2020-03-14 | Post modified: 2020-03-14 | In database

安装mysql5.7

  1. 下载并安装Mysql官方的yum repo
    [root@localhost ~]# wget -i -c http://dev.mysql.com/get/mysql57-community-release-el7-10.noarch.rpm
    [root@localhost ~]# yum -y install mysql57-community-release-el7-10.noarch.rpm
  2. 开始安装mysql服务器
    [root@localhost ~]# yum -y install mysql-community-server
  3. mysql数据库配置
    启动mysql：
    [root@localhost ~]# systemctl start mysqld.service
    [root@localhost ~]# systemctl status mysqld.service
    通过如下命令在日志文件中找出密码：
    [root@localhost ~]# grep "password" /var/log/mysqld.log
    进入mysql，修改密码和初始密码规则： 修改编码：
    在[mysqld]标签下加上以下内容：
    character_set_server = utf8
    在[client]标签下加上以下内容：
    default-character-set = utf8
    重启mysql，查看编码：
  4. 卸载yum repo
    [root@localhost ~]# yum -y remove mysql57-community-release-el7-10.noarch

    配置mysql主从

  5. 配置主库，在master上操作
  6. 配置从库，在slave上操作
  7. 重启两台服务器上mysql服务
    systemctl restart mysqld
  8. 在主服务器上建立用户授权slave
    用root用户登陆，然后执行如下操作：
    GRANT REPLICATION SLAVE ON *.* to 'slaveuser'@'%' identified by 'spdb1234';
  9. 登陆主服务器bdpkylin01，查询Master状态
    show master status;
    结果如下，即为正常
    File Position Binlog_Do_DB Binlog_Ignore_DB Executed_Grid_Set
    mysql-bin.00001 323
  10. 配置从服务器的slave，用root用户在bdpkylin02上操作
    change master to master_host='bdpkylin01’,master_user='slaveuser',master_password='spdb1234', master_log_file='mysql-bin.000001',master_log_pos=323;
    注意：master_log_pos=323 应为master position 值
  11. 启动从服务器复制功能，在bdpkylin02上操作
    start slave;
  12. 检查从服务器复制功能状态，在bdpkylin02上操作
    show slave status \G;
    需确保以下两个指标均为Yes，否则复制功能出现问题。
    Slave_IO_Running: Yes Slave_SQL_Running: Yes

与Hive的集成

表的数据存储在hbase中，元数据存储在hive中
两种方式

  • 管理表：创建hive表的时候，指定数据存储在hbase中，此时删除hive表，对应的hbase表也会删除
  • 外部表：现在已经存在Hbase表，需要对表中的数据进行分析

    集成配置

    1）修改hive-site.xml配置文件，添加配置属 2）修改hive-env.sh文件 3）创建hbase表及插入数据 4）创建并关联hive表 5）查询hive表，查看hbase中插入的数据
    select * from hbase_hive
    6）从hive表中插入数据
    insert into hbase_hive select rowkey, name from test;
    7）查询hbase，查看hive中插入的数据

    应用场景

  • 日志文件利用正则表达式，存储在hive表中
  • 创建hive和hbase关联表，将源日志表中的数据加工后
  • insert到hive和hbase的关联表中
  • 使用hbase进行交互式查询

Hbase核心—compact介绍

Posted on 2020-03-13 | Post modified: 2020-03-14 | In bigdata

Compact类型

minor合并

将多个小文件从写为数量较少的大文件，多路归并算法，只受磁盘IO的性能影响。Storefile随着Memstore被刷新越来越多，当Storefile的数量满足一定的条件时，会触发minor compact操作，合并成一个大Storefile文件，当文件大小超过单个文件配置的最大限制时，触发region的自动分割。

major合并

将一个region中一个列簇的若干个HFile重写为一个新的HFile。同时，含有更独特的功能，能扫描所有的键/值对，顺序重写全部数据，重写 过程略过删除标记的数据、超过版本号限制的数据、生存时间到期的数据。该操作属于重量级compact，会阻塞所有客户端对该操作所属region的请求，直到合并结束。

Posted on 2020-03-13 | Post modified: 2020-03-14 | In bigdata

FilterList

FilterList 代表一个过滤器列表，可以添加多个过滤器进行查询，多个过滤器之间的关系有：
与关系（符合所有）：FilterList.Operator.MUST_PASS_ALL
或关系（符合任一）：FilterList.Operator.MUST_PASS_ONE
使用方法：

过滤器的种类

  • 列值过滤器—SingleColumnValueFilter：过滤列值的相等、不等、范围等
  • 列名前缀过滤器—ColumnPrefixFilter：过滤指定前缀的列名
  • 多个列名前缀过滤器—MultipleColumnPrefixFilter：过滤多个指定前缀的列名
  • rowKey 过滤器—RowFilter：通过正则，过滤 rowKey 值。

    列植过滤器 —SingleColumnValueFilter

    SingleColumnValueFilter 列值判断:
  • 相等 (CompareOp.EQUAL ),
  • 不等(CompareOp.NOT_EQUAL),
  • 范围 (e.g., CompareOp.GREATER)
    下面示例检查列值和字符串’values’ 相等: 注意：如果过滤器过滤的列在数据表中有的行中不存在，那么这个过滤器对此行无法过滤。

    列名前缀过滤器 —ColumnPrefixFilter

    ColumnPrefixFilter 用于指定列名前缀值相等。

    多个列值前缀过滤器 —MultipleColumnPrefixFilter

    MultipleColumnPrefixFilter 和 ColumnPrefixFilter 行为差不多，但可以指定多个前缀。

    rowKey过滤器—RowFilter

    RowFilter 是 rowkey 过滤器 通常根据 rowkey 来指定范围时，使用 scan 扫描器的 StartRow 和 StopRow 方法比较好。
