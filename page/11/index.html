Hbase基础—系统特性

Posted on 2020-03-12 | Post modified: 2020-04-23 | In bigdata

Hbase系统特性

Hbase的优点

  • 容量巨大：Hbase单表可以支持千亿行、百万列的数据规模，数据容量可以达到TB甚至PB级别。
  • 良好的可扩展性：Hbase集群可以非常方便的实现集群容量扩展，包括数据存储节点以及读写服务节点扩展。
  • 稀疏性：Hbase支持大量稀疏存储，即允许大量列值为空，并且不占用任何存储空间。
  • 高性能：Hbase目前主要擅长OLTP场景，数据写操作性能强劲，对于随机单点读及小范围的扫描读，其性能也能得到保证。对于大范围的扫描读可以使用MR提供的API，以便实现更高效的并行扫描。
  • 多版本：Hbase支持多版本，即一个KV可以同时保留多个版本。
  • 支持过期：Hbase支持TTL过期特性。
  • Hadoop原生支持：

    Hbase的缺点

  • Hbase本身不支持很复杂的聚合运算，如Join、GroupBy等。如果业务需要，可以在Hbase之上架设Phoenix组件或者Spark组件，前者适合小规模聚合的OLTP场景，后者应用与大规模聚合的OLAP场景。
  • Hbase本身并没有实现二级索引功能，可以使用Phoenix提供的二级索引功能。
  • Hbase原生不支持全局跨行食物，只支持单行事务模型，同样可以使用Phoenix提供的全局事务模型组件来进行完善。

为啥用raid

RAID0

数据库事务

事务是逻辑上的一组操作，要么都执行，要么都不执行。

事物的四大特性(ACID)

  1. 原子性（Atomicity）：事务是最小的执行单位，不允许分割。事务的原子性确保动作要么全部完成，要么完全不起作用；
  2. 一致性（Consistency）：执行事务前后，数据保持一致，多个事务对同一个数据读取的结果是相同的；

快捷键 作用
command + option + 鼠标 列编辑
command + F 在当前文件中查找
command + W 关闭当前sheet
command + option + F 替换
command + shift + F 在打开的所有文件中查找
command + N 创建一个新的sheet
command + L 选中当前行，重复按下将下一行加入选
command + b 快速定位到变量的来源
control + [ 向左缩进
command + ] 向右缩进
command + control + ↑↓ 上下移动当前行

command + K + L 将字母改为小写
command + K + K 从光标处删除至行尾
command + K + backspace 从光标处删除至行首
shift + control + K 删除当前行
command + J 合并多行
command + option + 1234 将窗口分割成1、2、3、4列
command + enter 在当前行后插入新行
