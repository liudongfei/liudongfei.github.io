git的四部分构成

git常用命令

初始化仓库：git init
从远程仓库克隆项目到本地：
git clone git@github.com:git帐号名/仓库名.git

拉取远程分支到本地仓库：
git checkout -b 本地分支 远程分支 # 会在本地新建分支，并自动切换到该分支
git fetch origin : 远程分支 本地分支 # 会在本地新建分支，但不会自动切换，还需checkout
git branch –set-upstream 本地分支 远程分支 #建立本地分支与远程分支之间的连接

将文件添加都暂存区：
git add 文件名 #将工作区的某个文件添加到暂存区
git add -u #添加所有被tracked文件中被修改或删除的文件信息暂存区，不处理untracked的文件
git add -A #添加所有被tracked文件中被修改或删除的文件信息暂存区，包括untracked的文件
git add . #将工作区中的所有文件添加到暂存区
git add -i #进入交互界面，按需添加文件到缓存区

将暂存区的文件提交到本地仓库：
git commit -m “提交说明“ #将暂存区的文件提交到本地仓库
git commit -a -m “提交说明“ #跳过缓存区操作，直接将工作区的所有文件提交到本地仓库

查看仓库当前状态：
git status

比较文件异同：
git diff #工作区和暂存区的差异
git diff 分支名 #工作区与某分支的差异，远程分支这样写：remotes/origin/分支名
git diff HEAD #工作区与HEAD指针指向内容的差异
git diff 提交id 文件路径 #工作区某文件当前版本和历史版本的差异
git diff –stage #工作区文件和上次提交的差异，（1.6版本前用–cached）
git diff 版本TAG #查看某个版本后的改动内容
git diff 分支A 分支B #比较分支A和B的差异（也支持比较TAG）

查看历史记录：
git log #查看所有commit记录(SHA-A校验和，作者名称，邮箱，提交时间，提交说明)
git log -p -次数 #查看最近多少次的提交记录
git log –stat #简略显示每次提交的内容修改
git log –name-only #仅显示已修改的文件清单
git log –name-status #显示新增，修改，删除的文件清单
git log –oneline #让提交的文件以一行输出
git log -graph -all –oneline #图形展示分支的合并历史
git log –author=作者 #查询作者的提交记录（和grep使用时要加–all-match参数）
git log –grep=过滤信息 #列出提交信息中包含过滤信息的提交记录
git log filename #查看某文件的提交记录
代码回滚：
git reset HEAD^ #恢复成上次提交的版本
git reset HEAD^^ #恢复成上上次提交的版本，就是多个^，以此类推或者用 ～次数

同步本地仓库到远程仓库：
git push -u origin master
git push origin localBranch:remoteBranch 从本地的指定分支推送到远程指定分支
git pull origin remoteBranch:localBranch 从远程的指定分支拉取到本地指定分支

本地仓库内容推送到远程仓库：
git remote add origin git@github.com:帐号名/仓库名.git

分支管理：
git branch 查看本地分支
git branch -r 查看远程分支
git branch newBranchName 创建新的本地分支，不切换
git branch -b newBranchName 创建新的本地分支，切换到新建的分支
git checkout branchName 切换到指定的分支
git branch -d branchName 删除合并过的分支
git branch -D banchName 强制删除分支
git merge branchName 将指定的分支合并到当前分支

git push origin localBranch 将本地分支推送到远成分支
git push origin :remoteBranch 删除远程分支

查看分支合并图：
git log –graph –pretty=oneline –abbrev-commit

查看远程仓库地址：
git remote
// -v 显示更详细的信息

撤销某次提交：
git revert HEAD #撤销最近的一个提交
git revert 版本号 #撤销某次commit

打标签：
git tag 标签 #打标签命令，默认为HEAD
git tag #显示所有标签
git tag 标签 版本号 #给某个commit版本添加标签
git show tag #显示某个标签的详细信息
忽略文件及文件夹：
在仓库根目录下创建名称为“.gitignore”的文件，写入不需要的文件夹名或文件，每个元素占一行即可。

hadoop核心技术（二）—hdfs元数据管理剖析

Posted on 2019-05-14 | Post modified: 2020-03-11 | In bigdata

##概述
Hdfs的元数据存储在namenode的内存中，为了防止namenode宕机导致所有的元数据丢失，需要定期备份内存中的数据到硬盘，但如果元数据内容过大，则在dump时会耗费很长时间并且会影响hdfs的性能，因此出现了secondnamenode的角色。二者配合进行元数据管理。
##元数据管理
元数据管理流程图：

流程：

  1. Client端跟nn（namenode）交互会造成nn元数据的更新，同时基于WAL思想，所有涉及到更新元数据的操作都会写入编辑日志中（edits-in-progress文件）。
  2. snn（secondarynamenode）的进程会定期向nn的进程发起合并edits和fsimage的请求。
    1. nn根据时间和edits文件大小进行判断是否需要合并
    2. 如果需要进行合并则先触发edits文件（图中的edits-in-progress-0009-000X）的回滚操作，生成需要合并的edits文件（edits-0009-0012）和新的edits文件（edits-in-progress-0013-000X）
    3. nn将需要合并的edits文件（edits-0009-0012）发送到snn
    4. snn将上一次dump的fsimage（fsimage-0008）加载到内存中，然后将收到的edits文件（edits-0009-0012）依次加载到内存中
    5. snn将内存中的元数据dump成新的fsimage（fsimage-0012）写入磁盘
    6. Snn将新的fsimage发送给nn，nn接收成功后清除旧的fsimage和edits文件

hadoop核心技术（一）—hdfs读写流程

Posted on 2019-05-14 | Post modified: 2020-03-11 | In bigdata

hdfs写数据文件流程

hdfs写入数据流程图如下：

流程：

  1. client连接namenode存数据文件。
  2. namenode记录一条数据位置信息（元数据，一个元数据大概150b），告诉client存哪里。
  3. client用hdfs的api将数据块（默认128M）存储到datanode上（选择最近的一个datanode建立网络连接）。
  4. datanode将数据水平备份（客户端决定该文件的副本数量）。并且备份完成反馈给client。
  5. client通知namenode存储块完毕。
  6. namenode将元数据同步到内存中。
  7. 另一块重复上面的过程。

快捷键 作用
command + d 快速复制并在下一行插入当前行
command + / 注释当前行的代码
command + y 删除当前行的代码
command + p 方法参数提示
command + j 快速自动生成一些自动代码，如for循环
command + b 快速定位到变量的来源
control + h 查看类的继承关系
command + o 根据类名全局查看类
command + shift + 8 列编辑模式
command + shift + ↑↓ 将选中的代码上下移动
command + option + l 格式化代码
command + option + t 把选中的代码放到try{}if{}里
command + option + b 导航到一个抽象方法的实现代码
shift + enter 切换到下一行
option + enter 快速导入包
control + enter 新建界面或getter，setter界面
control + o 快速覆写父类中的方法
