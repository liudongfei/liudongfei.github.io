Linux运维—Linux的由来

Posted on 2022-02-27 | Post modified: 2022-03-07 | In linux | 浏览 次

Linux之前UNIX的历史

  • 1969年以前，输入设备只有卡片阅读机，输出设备只有打印机，用户无法与操作系统进行互动。后来设备更新和操作系统的改良，可以使用键盘进行输入。此时，如果多人使用一台主机，则需要等待。后来麻省理工学院开发了分时操作系统，可以使大型主机通过提供多个终端连接进入主机，利用主机资源进行运算工作，但是终端只具备输入/输出的功能，本身不具有任何运算或软件安装的能力，同时比较先进的主机也只能链接30台左右的终端机。后来为了增加大型主机的功能，在1965年前后由贝尔实验室、麻省理工学院及通用电气公司共同发起了Multics项目，为了让大型主机同时提供300台以上的终端连接机使用的目的。在1969年前后，由于项目进度落后，资金短缺，贝尔实验室推出了该项目。
  • 1969年，原本参与Multics项目的成员Thompson，为了自己需要，以汇编语言(Assemler)写出来一组内核程序，同时包括一些内核工具程序及一个小的文件系统。该系统就是UNIX的原型。
  • 1973年，UNIX正式诞生。由于Thompson写的那个操作系统太好用了，在贝尔实验室内部广为流传，但是，由于是由汇编语言编写的，和计算机架构的兼容性太差，因此Thompson和Ritchie合作用C语言重新改写和编译，最后发行出了UNIX的正式版本。
  • 1977年，重要的UNIX分支—BSD诞生。贝尔实验室属于AT&T公司，对于UNIX采取较为开放的态度，在1973年后UNIX便得以与学术界合作开发，最重要的就是与加州柏克莱（Berkeley）大学合作。柏克莱大学的BillJoy在取得UNIX的源码后，着手修改为舍和自己机器的版本，同时增加了很多工具软件与编译程序，最终将它命名为Berkeley Software Dsitribution（BSD）。这个BSD是UNIX很重要的一个分支，Bill Joy也是Sun这家公司的创办者。Sun公司即是以BSD开发的内核进行自己的商业UNIX版本的开发的。
  • 1979年，由于UNIX的高度可移植性与强大的性能，加上当时并没有版权的纠纷，所以很多公司开始了UNIX操作系统的开发，例如AT&T的System V、IBM的AIX等。而在早期每家生产计算机硬件的公司还没有“协议”的概念，所以每一家计算机公司生产的硬件自然就不相同，因此需要为自己的计算机硬件开发合适的UNIX系统，而且仅能与自身的硬件支持，这种情况在1979年AT&T推出System V第七版UNIX后有所改善，这个版本的System V支持x86架构的个人计算机。由于商业原因，AT&T在第七版的UNIX中，特别提到“不可对学生提供源码”的严格限制
  • 1984年，Richard Mathew Stallman（史托曼）发起了GNU项目，对于当今自由软件风潮具有不可磨灭的地位。这个项目是创建一个自由、开放的UNIX操作系统(Free UNIX)，著名的GNU C Compiler（GCC）、Emacs以及常用的C函数库、bash shell等都是他完成的。为了避免GNU开发的自由软件被其他所利用成为专利软件，他与律师草拟了有名的通用公共许可证（GPL）。
  • 1991年，芬兰的赫尔辛基大学的Linus Torvalds以bash，gcc等工具写了一个小小的内核程序，这个内核程序可以在Intel的386机器上运行。

Linux的内核版本

  • 主、次版本为奇数：开发中版本。主要在测试和开发新功能。
  • 主、次版本为偶数：稳定版本

Linux的Distributions

  • Linux的Distributions包括“Kernel + Software + Tools”
  • 一般Distribution主要分为两大系统，一种使用RPM方式安装软件的系统，如Red Hat，Fedora，SuSE等；一种则是使用Debian的dpkg方式，如Debian，Ubuntu，B2D等。
  • 如何选择Distribution？如果装在个人计算机上，建议使用团队版Fedora，Ubuntu，OpenSuSE等。如果用在服务器上建议使用商业版本，如Red Hat，SuSE。
