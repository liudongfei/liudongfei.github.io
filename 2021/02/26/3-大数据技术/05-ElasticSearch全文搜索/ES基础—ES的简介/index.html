ES基础—ES的简介

Posted on 2021-02-26 | Post modified: 2021-02-26 | In database | 浏览 次

ES的基本概念

  • 索引(Index): 索引是ES存放具体数据的地方，是一类具有相似特征的文档集合。ES中的索引具有不同的意思，这里的索引相当于关系型数据库的一个实例。在ES中还可以作为动词，表示对文档数据进行索引操作。
  • 类型(Type): 在6.0版本之前，在一个ES的索引中可以有多个类型，从6.0版本开始，一个索引中，只有一个类型。ES的类型概念相当于关系数据库中的数据表。在同一个索引中，不同映射类型中具有相同名称的字段在内部由相同的Lucene字段支持，会产生问题的。
  • 文档(Doc): 文档是ES可被索引的基础单元，相当于关系数据库中数据表的一条数据。ES的文档具有Json格式，由多个字段组成，字段相当于关系数据库中列的概念。
  • 映射(Mapping): ES用映射来定义一个文档，可以定义所包含的字段以及字段的类型、分词起及属性等。
  • 分片(shard): 当数据量较大时，索引的存储空间需求超出单个节点磁盘容量的限制，或者出现单个节点处理速度较慢。为了解决这些问题，ES将索引中的数据进行切分成多个分片（shard），每个分片存储这个索引的一部分数据，分布在不同节点上。当需要查询索引时，ElasticSearch将查询发送到每个相关分片，之后将查询结果合并，这个过程对ES应用来说是透明的，用户感知不到分片的存在。 一个索引的分片一旦指定，不能再修改。
  • 副本(replicas): 分片全称是主分片，简称为分片。主分片是相对于副本来说的，副本是对主分片的一个或多个复制版本（或称拷贝），这些复制版本（拷贝）可以称为复制分片，可以直接称之为副本。当主分片丢失时，集群可以将一个副本升级为新的主分片。

    ES的映射

    映射的类型

    映射可以分为动态映射、静态映射和动态+静态三种类型。
  • 动态映射*
    这种机制下，不需要在ES的索引中事先定义映射，文档写入ES会根据字段自动识别类型。动态映射规则如下：
Json数据 自动推测的类型
null 没有字段被添加
true或者false boolean型
小数 float型
数字 long型
日期 date或text
字符串 text
数组 由数组的第一个非空值决定
Json对象 object类型
静态映射
这种机制下，在ES的索引中事先定义好映射，在进行文档插入。
动态+静态
首先，预先定义好映射，在插入文档时，映射中存在的属性采用静态映射确定属性类型，映射中不存在的属性采用动态映射自动推测类型
#### 映射元数据和参数
元数据 说明
— —
_index 文档所属的索引
_id 文档的id
_type 文档所属类型
_source 文档的原生json字符串
_size 整个_source字段的字节数大小
_field_names 索引了每个字段的名称
_routing 将一个文档根据路由存储到指定的分片上
_meta 用于自定义元数据

映射(mapping)参数

  • analyzer：指定分词器
  • normalizer：用于解析前的标准化配置，如把所有的字母转换成小写
  • boost：通过boost值来控制查询子句的相对权重，该值默认为1。一个大于1的boost会增加该查询子句的相对权重
  • coerce：用于清除脏数据，默认值true。字符串会被强制转换为整数，浮点数被强制转换为整数。
  • dynamic：用于检测新发现的字段，有三个取值：true，新发现的字段添加到映射中；false，新检测的字段被忽略，必须显式添加新字段。strict，如果新检测到字段，就会引发异常并拒绝文档。
  • enabled：ES默认会索引所有字段，enabled设置为false的字段，es会跳过字段内容，该字段只能从_source字段中获取，但是不可搜索。
  • format：指定字段值的格式
  • ignore_above：指定字段索引和存储的长度最大值，超过最大值的会被忽略

映射的数据类型

分类 具体类型
字符串类型 string,text,keyword
整数类型 integer,long,short,byte
浮点类型 double,float,half_float,scaled_float
逻辑类型 boolean
日期类型 date
范围类型 range
二进制类型 binary
数组类型 array
对象类型 integer,long,short,byte
嵌套类型 double,float,half_float,scaled_float
